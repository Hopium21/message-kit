# AI Skills

MessageKit's AI Skills system enables bots to understand and process skills through multiple intelligence layers, creating more natural and powerful conversational experiences.

## Introduction to AI Skills

## 1. Command Processing

Bots can execute structured skills with parameter validation and type checking:

```jsx
// Execute a simple command
const { message } = await context.skill("/send 10 usdc to @alix", conv);
context.send(message);
```

Learn more about [command processing](./commands.mdx).

## 2. Natural Language Understanding

Bots can interpret casual human language and convert it to executable skills:

```bash
User: "@swap 1 usdc to eth please"
Bot: *converts to* "/swap 1 usdc eth"
```

Learn more about [natural language understanding](./natural.mdx).

## 3. Multi-Step Reasoning

Bots can manage complex workflows by:

- Breaking down tasks into steps
- Maintaining conversation context
- Executing sequential commands
- Waiting for user input when needed

Example of a multi-step ENS registration flow:

```tsx
["Let's first check if the domain is available.  ", "/check fabriethsf.eth"];
["The domain is available!  ", "/register fabriethsf.eth"];
["The domain is registered!  ", "/info fabriethsf.eth"];
```

Learn more about [multi-step reasoning](./reasoning.mdx).

## 4. Multi-Agent

Multiple specialized bots can work together in the same chat, each handling different aspects of user requests.

Learn more about [multi-agent](./multi-agent.mdx).

## Setting Up Skills

### 1. Define skills

Create a `skills.ts` file with your command definitions:

```tsx
export const skills: SkillGroup[] = [
  {
    name: "Transaction Bot",
    description: "Handles crypto transactions",
    skills: [
      {
        command: "/send [amount] [token] [username]",
        triggers: ["/send"],
        examples: ["/send 10 usdc @vitalik"],
        description: "Send crypto to a user",
        handler: transactionHandler,
        params: {
          amount: { type: "number", default: 10 },
          token: {
            type: "string",
            values: ["eth", "usdc", "dai"],
          },
          username: { type: "username" },
        },
      },
    ],
  },
];
```

### 2. Create Command Handlers

Implement the logic for each command:

```tsx
async function transactionHandler(context: HandlerContext) {
  const { command, params } = context.message.content;
  const { amount, token, username } = params;

  // Execute transaction logic
  const result = await processTransaction(amount, token, username);

  return {
    message: `Transaction complete! Sent ${amount} ${token} to ${username}`,
  };
}
```

### 3. Add Natural Language Support

Define prompts that help your bot understand user intent:

```tsx
const systemPrompt = `
You are a helpful crypto transaction bot that:
- Understands natural requests about sending/swapping tokens
- Converts requests into proper commands
- Only uses supported tokens: eth, usdc, dai
- Maintains a friendly, professional tone

Command format:
/send [amount] [token] [username]
/swap [amount] [token_from] [token_to]
`;
```

## Best Practices

1. **Validation**: Always validate parameters before execution
2. **Error Handling**: Provide clear error messages when skills fail
3. **Examples**: Include usage examples in command definitions
4. **Progressive Enhancement**: Start with basic commands, then add natural language support
5. **Context Awareness**: Maintain conversation state for multi-step operations

## Advanced Features

- **Custom Parameter Types**: Define your own parameter validators
- **Chained Commands**: Execute multiple skills in sequence
- **Conditional Logic**: Add branching based on user responses
- **State Management**: Track conversation context across multiple messages

This improved documentation provides a clearer structure, more practical examples, and better explains the capabilities of the AI Skills system while maintaining technical accuracy.
